# قهوتك | Qahuatk

واجهة قائمة مشروبات تفاعلية مبنية باستخدام **Vite + JavaScript + SCSS**. المشروع يعرض تصنيفات المشروبات، تفاصيل كل منتج، أسعاراً حسب الحجم، إضافات اختيارية، وفاتورة ديناميكية يمكن التحكم بعناصرها.

## المميزات

- واجهة عربية باتجاه RTL.
- تصميم متجاوب للهاتف والتابلت وسطح المكتب.
- نافبار علوية ثابتة للحاسوب، ونافبار سفلية ثابتة للتابلت والموبايل.
- قائمة مشروبات مبنية من ملف JSON.
- صور محلية يتم تضمينها في البناء عبر Vite.
- صفحة منتج تعرض الصورة، السعر، المكونات، الأحجام، والإضافات.
- أسعار مختلفة حسب الحجم:
  - صغير: أقل من السعر الأساسي.
  - وسط: السعر الأساسي.
  - كبير: أعلى من السعر الأساسي.
- فاتورة ديناميكية تدعم:
  - إضافة العناصر.
  - زيادة الكمية.
  - إنقاص الكمية.
  - حذف العناصر.
  - حساب الضريبة والإجمالي.
- تأثيرات hover للبطاقات والأزرار مع استثناء النافبار من الوميض.

## التقنيات

- Vite
- JavaScript ES Modules
- SCSS / Sass
- LocalStorage لحفظ عناصر الفاتورة داخل المتصفح
- Material Symbols للأيقونات

## تشغيل المشروع

ثبت الاعتماديات:

```bash
npm install
```

شغل خادم التطوير:

```bash
npm start
```

أو:

```bash
npm run dev
```

افتح الرابط:

```text
http://127.0.0.1:5173/
```

## أوامر مفيدة

```bash
npm start
```

تشغيل المشروع محلياً.

```bash
npm run build
```

بناء نسخة الإنتاج داخل مجلد `dist`.

```bash
npm run preview
```

معاينة نسخة الإنتاج بعد البناء.

```bash
npm audit
```

فحص الاعتماديات أمنياً.

## النشر على GitHub Pages

المشروع يستخدم Vite، لذلك يجب نشر نسخة `dist` الناتجة من `npm run build` وليس ملفات المصدر مباشرة.

تم ضبط `vite.config.js` على:

```js
base: "/qahuatk/"
```

هذا ضروري لأن رابط GitHub Pages للمشروع يعمل داخل مسار فرعي:

```text
https://muhammet-g.github.io/qahuatk/
```

يوجد workflow جاهز في:

```text
.github/workflows/deploy.yml
```

بعد الدفع إلى `main`، سيقوم GitHub Actions بتثبيت الاعتماديات، بناء المشروع، ثم نشر مجلد `dist` على GitHub Pages.

من إعدادات المستودع في GitHub تأكد أن:

```text
Settings > Pages > Build and deployment > Source = GitHub Actions
```

## مصدر البيانات

الملف الأساسي للبيانات هو:

```text
src/data/drinks.json
```

يحتوي على:

- `categories`: تصنيفات المشروبات.
- `products`: المنتجات، وتشمل الاسم، السعر، التصنيف، الصورة، والمكونات.

مثال منتج:

```json
{
  "id": "latte",
  "name": "لاتيه",
  "category": "hot_drink",
  "price": 22,
  "image": "../asesset/images/hot_drink/Latte.png",
  "components": ["إسبريسو", "حليب مبخر", "رغوة حليب ناعمة"]
}
```

ملف `src/data/menu.js` يقرأ JSON ويحوّل مسارات الصور إلى أصول صالحة داخل Vite باستخدام `import.meta.glob`.

## إضافة مشروب جديد

1. أضف صورة المنتج داخل التصنيف المناسب في:

```text
src/asesset/images/
```

2. أضف المنتج داخل `products` في:

```text
src/data/drinks.json
```

3. تأكد أن `category` يطابق أحد معرفات التصنيفات.

4. شغل:

```bash
npm run build
```

## نظام الأسعار

السعر الموجود في `drinks.json` هو سعر الحجم الوسط.

يتم توليد أسعار الأحجام داخل:

```text
src/data/menu.js
```

القيم الحالية:

- `S`: السعر الأساسي - 3
- `M`: السعر الأساسي
- `L`: السعر الأساسي + 4

مع حد أدنى للسعر حتى لا تنخفض المشروبات الرخيصة بشكل غير منطقي.

## الفاتورة

الفاتورة تعتمد على `localStorage` بالمفتاح:

```text
qahuatk-cart
```

العنصر المحفوظ يحتوي على:

- المنتج
- الحجم
- الإضافات
- الكمية
- سعر الوحدة بعد الحجم والإضافات

## ملاحظات تطوير

- لا تعدّل ملفات `dist` يدوياً، لأنها تنتج من `npm run build`.
- الصور كبيرة نسبياً، لذلك يمكن لاحقاً تحسين الأداء بضغط الصور أو تحويلها إلى WebP.
- اسم مجلد الصور الحالي هو `asesset` كما هو موجود في المشروع. إذا تم تغييره إلى `assets` يجب تحديث المسارات في `drinks.json` و`menu.js`.
